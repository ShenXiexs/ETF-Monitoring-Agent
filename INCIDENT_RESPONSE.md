# Incident Response

## 1. 确认

- 确认问题是否真实存在
- 判断是否涉及密钥、上传链路、文档导出或服务暴露
- 记录受影响版本、配置和数据目录

## 2. 处置

- 若涉及密钥，先轮换密钥并检查仓库历史
- 若涉及文件上传或接口暴露，先停用对应入口或把服务改回 `127.0.0.1`
- 若涉及输出错误，先暂停使用生成内容并回滚到人工流程

## 3. 修复

- 补充代码修复
- 增加回归测试
- 更新 README 或运行文档，避免同类误用再次出现

## 4. 复盘

- 记录触发原因
- 记录修复时间线
- 明确是代码缺陷、配置失误还是数据问题
- 把后续动作沉淀到文档或默认配置里
